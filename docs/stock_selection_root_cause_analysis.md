# Stock Selection Root Cause Analysis

## Overview
This document analyzes the logical failure of the deterministic Sector → Stock mapping layer. While the MVP was halted at Sector Discovery, the logic governing the Sector mapping mathematically guarantees failure at the Stock level.

---

## TASK 1: Why Sector → Stock Fails

**The Process:**
Macro Input → Tags → Sector → Stock

**The Failure Trace:**
1. The macro event triggers a generic tag (e.g., `margin_expansion`).
2. The tag maps blindly to a broad sector (e.g., `Software & SaaS`).
3. The sector maps blindly to a static list of stocks (e.g., `[MSFT, CRM, SNOW]`).

**Why it fails:**
Reasoning disappears entirely at the Tag level. A macro event might be "New AI models drastically reduce software margins." The system sees the word "margin", triggers the `margin_expansion` tag, recommends the `Software` sector, and recommends buying `SNOW`.
The system took a headline specifically stating that software margins are *crashing*, and recommended buying a software stock because the dictionary saw the word "margin." The Sector → Stock layer fails because it inherits fundamentally corrupted, context-blind data from the ontology layer.

## TASK 3: Smallest Possible Constraint Layer

**Design: The Hard Filter Matrix**

Without LLMs or AI, the system can use boolean constraints to filter the final static stock list. We add a `metadata` object to every hardcoded stock.

* **Step 1:** The Framework generates a list of `Active_Constraints` (e.g., `["avoid_psu", "avoid_high_debt", "require_founder"]`).
* **Step 2:** The Sector map generates a raw list of stocks.
* **Step 3:** A simple Python loop checks the stock's static metadata against the `Active_Constraints`.

```python
# Minimal Constraint Logic
filtered_stocks = []
for stock in raw_sectors_stocks:
    reject = False
    for constraint in active_constraints:
        if constraint == "avoid_psu" and stock.metadata.get("is_psu") == True:
            reject = True
        if constraint == "avoid_high_debt" and stock.metadata.get("debt_to_equity", 0) > 2.0:
            reject = True
    if not reject:
        filtered_stocks.append(stock)
```

## TASK 4: Mental Re-Run of the 10 Tests

If we apply the Hard Filter Matrix to the 10 macro tests:

* **Expected Reduction in Failures:** **30% - 40%**.
* **Why:** The constraint layer successfully prevents catastrophic, rule-breaking errors. If Akshat's principle says "Avoid PSUs," the filter guarantees a PSU won't appear on the list, even if the sector is booming (e.g., Defense Indigenization).
* **Why failures remain:** It does not fix the false positives generated by the upstream ontology. If the system wrongly thinks "Software" is the primary sector for a "Semiconductor" bill, the constraint layer will simply output the "highest quality, low-debt Software stocks." It executed the constraint perfectly on the wrong sector.

## TASK 5: Can a constraint layer solve the stock problem?

**Answer:** NO. Deeper reasoning is required.

**Evidence:**
A constraint layer is a *negative* filter. It is very good at answering the question: "Is this stock legally allowed to be bought under my rules?"

However, opportunity discovery requires a *positive* filter. The system must answer: "Is this stock actually benefiting from this specific macro event?"

A static metadata tag on a stock (`is_founder_led=True`) cannot determine if that founder's company will benefit from a sudden, unmapped geopolitical shift in supply chains. Without semantic understanding of the macro event's *impact* on the specific business model, the system will just output a generic list of "Good Stocks" rather than a targeted list of "Stocks Benefiting Right Now."

---

## FINAL VERDICT

**Choice: B. Static mappings are fundamentally doomed.**

**Defense:**
Static mappings operate under the assumption that the world is finite and categorizable into a permanent JSON dictionary. Financial markets are reflexive, infinite, and linguistic.

If we map "Capex" to "Industrial Stocks," what happens when a new technology (e.g., AI compute) turns "Software Stocks" into capex-heavy businesses? The static map fails.
If we map "Inflation" to "Commodity Stocks," what happens when inflation is driven by a labor shortage rather than a supply shock? The static map outputs the wrong sector.

Static mappings (Keyword → Tag → Sector → Stock) destroy the nuance of language at every jump. You cannot take a complex, highly contextual input, compress it through three layers of static dictionaries, and expect a highly nuanced, alpha-generating stock pick to emerge on the other side. You will only ever get an average, generic list.

The architecture must transition to semantic interpretation, or it will never discover alpha.
